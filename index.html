01 · Instalación

Tres líneas, un reproductor

Cargá el script, dejá un contenedor vacío en el DOM, y conectá con un token JWT. El widget arma el iframe, hace el handshake seguro y arranca la reproducción.

HTML
<script src="https://sound.clubf5.com/widget-embed.js"></script>
<div id="clubf5-player"></div>

<script>
  ClubF5Widget.connect('clubf5-player', {
    token: 'JWT_TOKEN_HERE' // lo pide tu backend, ver sección Token
  });
</script>

El token nunca se escribe en el DOM ni en la URL del iframe: viaja de tu página al widget por postMessage, un canal que no queda expuesto en el HTML ni en el historial del navegador.

02 · Modos de conexión

Elegí cómo entregás el token

Dos formas de conectar el widget. Usá la recomendada salvo que ya tengas el modo legacy integrado.

connect()
Recomendado

Token entregado por JavaScript vía postMessage. Habilita la auto-renovación del widget.

JS
ClubF5Widget.connect('clubf5-player', {
  token: miToken,
  theme: 'dark',
  height: '80px'
});
data-token
Legacy

Token como atributo del contenedor. Más simple, pero el token queda en el DOM y no se auto-renueva.

HTML
<div
  id="clubf5-player"
  data-token="JWT_TOKEN_HERE"
></div>

Con data-token, cuando el token expira el widget deja de autenticar y no hay ningún canal vivo para pedir uno nuevo — hay que recargar la página con un token fresco. Con connect(), el widget renueva la sesión solo mientras el iframe siga abierto.

03 · Obtener un token

El token lo emite tu backend, nunca el sitio de terceros

Tu propia app (ya autenticada) le pide a la API un token de corta duración específico para el widget. Ese es el único valor que se le pasa al script embebido — nunca la sesión larga ni el refresh token real del usuario.

HTTP
POST /api/Token/WidgetToken
Authorization: Bearer <tu_token_de_sesion>
Content-Type: application/json

{ "horas": 2 }  // opcional · 1–24 · default 2
Campo de la respuestaDescripción
access_tokenJWT a pasar en ClubF5Widget.connect({ token })
refresh_tokenInterno del widget — no hace falta manejarlo a mano, se renueva solo.

El usuario/cuenta usada para pedir el token tiene que ser de rol "Reproductor" — cualquier otro rol es rechazado por el widget con "Rol no autorizado" antes de intentar reproducir nada.

05 · API de JavaScript

window.ClubF5Widget

Cuatro métodos, expuestos globalmente apenas carga el script.

connect(id, options) → Promise
Conecta un widget de forma segura. id es el ID del contenedor o un HTMLElement. Devuelve la instancia una vez creado el iframe.
init()
Autoinicializa contenedores con data-token (modo legacy). Corre sola al cargar el DOM, normalmente no hace falta llamarla a mano.
destroy()
Destruye todas las instancias activas y remueve sus iframes del DOM.
getInstances() → Array
Devuelve las instancias activas actuales, cada una con su propio destroy() puntual.
06 · Seguridad

Qué protege esta integración

Pensado específicamente para vivir en sitios que vos no controlás.

01

El token nunca va en la URL

Viaja por postMessage, validado por nonce y origen — no queda en el historial del navegador ni en logs de servidor.

02

Vida corta, no la sesión completa

El widget usa un JWT de horas, no el token de sesión de un año. Si el sitio de terceros se ve comprometido, la ventana de exposición es acotada.

03

Renovación automática y transparente

Mientras el iframe siga abierto, el propio widget pide un token nuevo antes de que el actual expire — el sitio anfitrión no hace nada.

04

Iframe en sandbox

allow-scripts allow-same-origin allow-popups allow-forms — sin acceso a APIs sensibles del documento anfitrión.

05

Reintentos con límite

Ante una falla real de renovación (no transitoria), el widget deja de insistir con un token que ya sabe inválido y repite el handshake desde cero.

07 · Solución de problemas

Errores comunes

"Contenedor no encontrado: #id"
El id pasado a connect() no existe en el DOM en el momento de la llamada. Asegurate de llamar a connect() después de que el <div> ya esté renderizado.
"El token JWT ha expirado"
El script valida el exp del token antes de crear el iframe (con 60s de margen). Pedí un token nuevo a /Token/WidgetToken — no reutilices uno viejo entre cargas de página.
"Rol no autorizado"
El widget solo acepta tokens cuyo claim de rol sea Reproductor. Verificá con qué usuario/cuenta se está pidiendo el WidgetToken del lado del backend.
"Handshake timeout - falling back to URL token"
El iframe no recibió respuesta al postMessage en 30s (el parent no llamó a connect() a tiempo, o el origen no coincidió). El widget cae al modo legacy como último recurso.
"Token no válido. Este refresh ya ha sido utilizado."
El refresh token es de un solo uso: cada renovación invalida el anterior. Este error aparece si una respuesta de renovación se perdió en el camino. El widget lo detecta como error terminal y repite el handshake completo en vez de seguir reintentando con un refresh que ya no sirve.
"El usuario ha finalizado su sesión de forma manual"
Alguien cerró sesión explícitamente para ese usuario (logout), lo que invalida también los tokens del widget emitidos a su nombre. Hay que pedir un WidgetToken nuevo tras un login fresco.
